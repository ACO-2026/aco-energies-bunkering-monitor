📡 Live AIS Integration — MarineTraffic Setup
📡
HOW IT WORKS: Your dashboard calls a Netlify serverless function (/.netlify/functions/vessel-proxy) which securely forwards requests to MarineTraffic's API. This avoids browser CORS restrictions and keeps your API key private (stored only in Netlify, never in browser code). The Minerva Smart Filter and gap-finder scoring are applied automatically to all live results.
📋 STEP 1 — Get MarineTraffic API Key
1. Go to marinetraffic.com/en/ais-api-services
2. Register for an API account
3. Subscribe to the Expected Arrivals API (API ID: 4)
4. Copy your API key

Recommended plan: Basic or Standard
Key endpoint: Expected Arrivals at Port Said (ID: 1590) and Suez (ID: 3318)
🔑 STEP 2 — Add Key to Netlify
1. Log in to app.netlify.com
2. Click your site → Site Configuration
3. Click Environment Variables
4. Click Add a variable
5. Key: MARINETRAFFIC_API_KEY
6. Value: your API key here
7. Click Save → Redeploy site

⚠ Never put your API key in the HTML/JS code.
⚙ STEP 3 — Configure in Dashboard
1. Click ⚙ Settings in the header (top right)
2. Confirm your Netlify function URL
3. Set refresh interval (30–120 seconds)
4. Click Save & Test Connection
5. If successful, status badge turns LIVE AIS

The dashboard auto-refreshes and applies the Minerva filter to all live vessel data.
✅ What You Get When Live
• Real vessel ETAs at Port Said & Suez (48hr window)
• Vessel name, IMO, flag, operator, DWT pulled from AIS
• Auto-scored against Minerva exclusion database
• New-caller flag based on AIS history
• Estimated fuel type and quantity from vessel class
• Commission estimate per eligible stem
• Auto-refresh every 60 seconds
• Demo data shown when API unavailable
⚙ Dashboard Settings — AIS Configuration
Configure your MarineTraffic API connection. Your API key is stored locally in your browser and sent only to your Netlify serverless proxy — never exposed in public code.
Default: /.netlify/functions/vessel-proxy — only change if you renamed the function file.
Get your key from marinetraffic.com/en/ais-api-services — Expected Arrivals API (ID: 4). Your key is stored in your browser's localStorage, not in any code.
